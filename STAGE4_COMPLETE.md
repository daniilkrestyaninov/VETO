# Этап 4 - Real-time Лобби (ЗАВЕРШЕН)

## Что реализовано:

✅ **SessionRepository** - полная работа с сессиями и вариантами
  - Создание/получение сессий
  - Добавление/удаление вариантов выбора
  - Запуск рулетки (изменение статуса)
  - Supabase Realtime подписки на изменения

✅ **Riverpod провайдеры с Realtime**
  - `ActiveSessionProvider` - активная сессия группы с live-обновлениями
  - `SessionOptionsProvider` - варианты выбора с real-time синхронизацией
  - Автоматическая подписка/отписка от каналов

✅ **UI экрана лобби (GroupLobbyScreen)**
  - Отображение кода группы с кнопкой копирования
  - Список участников в реальном времени с veto-токенами
  - Список вариантов выбора с live-обновлениями
  - Добавление новых вариантов
  - Удаление своих вариантов
  - Кнопка запуска рулетки (с валидацией минимум 2 варианта)

## Функционал экрана лобби:

### 1. Код группы
- Отображается в рамке
- Копируется по клику
- Кнопка "Поделиться" в AppBar

### 2. Участники
- Real-time обновление списка
- Отображение роли (ВЛАДЕЛЕЦ)
- Количество veto-токенов у каждого
- Визуальное выделение владельца (желтая рамка)

### 3. Варианты выбора
- Real-time синхронизация между всеми участниками
- Добавление вариантов через текстовое поле
- Удаление только своих вариантов (кнопка X)
- Автоматическое создание сессии при добавлении первого варианта

### 4. Запуск рулетки
- Валидация: минимум 2 варианта
- Изменение статуса сессии на "spinning"
- Переход на экран рулетки (/session/:sessionId)

## Supabase Realtime:

Приложение подписывается на изменения:
- **sessions** - статус сессии (waiting → spinning → resolved)
- **options** - добавление/удаление вариантов
- **group_members** - вход/выход участников (через GroupProvider)

Все участники видят изменения мгновенно без перезагрузки!
