## База данных

### Схема
```sql
users (id, username, avatar_url, created_at)
groups (id, name, created_by, created_at)
group_members (id, group_id, user_id, role, veto_tokens, joined_at)
sessions (id, group_id, status, final_decision_id, created_at, resolved_at)
options (id, session_id, title, added_by, created_at)
veto_logs (id, session_id, user_id, reason, used_at)
```

### RLS (Row Level Security)
⚠️ **Текущее состояние**: Отключен для всех таблиц (для разработки)

**Причина**: Политики RLS создавали бесконечную рекурсию при проверке прав доступа.

**Решение для продакшена**: Включить RLS с упрощенными политиками:
```sql
-- Пример для group_members
CREATE POLICY "Enable read access for all users"
ON group_members FOR SELECT USING (true);

CREATE POLICY "Enable insert for authenticated users"
ON group_members FOR INSERT WITH CHECK (auth.uid() IS NOT NULL);
```

## Проблемы и решения

### 1. Бесконечная рекурсия в RLS ✅
**Проблема**: Политики RLS для `group_members` создавали рекурсию при проверке прав.

**Решение**: 
- Отключили RLS для разработки
- Создали SQL миграции для правильных политик
- Настроили Supabase CLI для управления базой

### 2. "Bad state: Future already completed" ✅
**Проблема**: Ошибка при навигации из-за `WidgetsBinding.instance.addPostFrameCallback`.

**Решение**: Заменили на `Future.microtask` в двух местах.

### 3. Проблемы с быстрым стартом ✅
**Проблема**: При повторном входе заходил под предыдущим аккаунтом.

**Решение**: Добавили выход из текущей сессии перед анонимным входом.

### 4. Добавление/удаление вариантов ✅
**Проблема**: Варианты не добавлялись/удалялись (казалось).

**Решение**: Добавили подробное логирование, которое показало, что функционал работает. Проблема была в задержках Realtime.

## Известные ограничения

1. **RLS отключен** - небезопасно для продакшена
2. **Нет восстановления пароля** - функция не реализована
3. **Нет истории вето в UI** - логи записываются, но не отображаются
4. **Нет профиля пользователя** - нельзя изменить имя/аватар
5. **Нет обработки сетевых ошибок** - при потере интернета приложение может зависнуть
6. **Быстрый старт имеет проблемы** - рекомендуется использовать полную регистрацию
