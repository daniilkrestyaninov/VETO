**Файл**: `lib/features/session/data/session_repository.dart`

**Добавленные методы**:
- `getUserVetoTokens()` - получение количества токенов вето пользователя
- `useVeto()` - использование вето (транзакция):
  1. Проверка количества токенов
  2. Проверка статуса сессии (должна быть resolved)
  3. Списание токена вето
  4. Запись в лог вето
  5. Отмена решения (изменение статуса сессии на waiting)
- `getVetoLogs()` - получение истории вето для сессии
- `subscribeToVetoLogs()` - подписка на изменения логов вето через Realtime

**Логика транзакции**:
```dart
// 1. Проверяем количество токенов
final tokens = await getUserVetoTokens(groupId: groupId, userId: userId);
if (tokens <= 0) throw Exception('У вас нет токенов вето');

// 2. Проверяем статус сессии
final session = await getSession(sessionId);
if (session.status != 'resolved') throw Exception('Вето можно использовать только на завершенную сессию');

// 3. Списываем токен
await _supabase.from('group_members')
    .update({'veto_tokens': tokens - 1})
    .eq('group_id', groupId)
    .eq('user_id', userId);

// 4. Записываем в лог вето
await _supabase.from('veto_logs').insert(vetoLogData);

// 5. Отменяем решение
await _supabase.from('sessions')
    .update({
      'status': 'waiting',
      'final_decision_id': null,
      'resolved_at': null,
    })
    .eq('id', sessionId);
```

## Архитектура логики вето

### Поток данных
```
1. Пользователь нажимает "ИСПОЛЬЗОВАТЬ ВЕТО"
   ↓
2. Показывается диалог подтверждения
   ↓
3. SessionScreen._handleVeto() вызывается
   ↓
4. VetoLogsProvider.useVeto() вызывается
   ↓
5. SessionRepository.useVeto() выполняет транзакцию:
   - Проверка токенов
   - Проверка статуса сессии
   - Списание токена
   - Запись в лог
   - Отмена решения
   ↓
6. Realtime уведомляет всех участников об изменениях
   ↓
7. Пользователь возвращается в лобби группы
```

### Realtime подписки
- **sessions** - изменения статуса сессии
- **options** - добавление/удаление вариантов
- **veto_logs** - новые записи о вето
- **group_members** - изменение количества токенов

## Тестирование функционала вето

### Сценарий 1: Успешное использование вето
1. Создать группу
2. Добавить 2+ вариантов
3. Запустить рулетку
4. Дождаться результата
5. Нажать "ИСПОЛЬЗОВАТЬ ВЕТО"
6. Подтвердить в диалоге
7. **Ожидаемый результат**: 
   - Токен вето списан (было 3, стало 2)
   - Сессия отменена (статус waiting)
   - Пользователь вернулся в лобби
   - Можно запустить рулетку снова

### Сценарий 2: Нет токенов вето
1. Использовать все 3 токена вето
2. Попытаться использовать вето снова
3. **Ожидаемый результат**: Ошибка "У вас нет токенов вето"

### Сценарий 3: Realtime уведомления
1. Открыть приложение на двух устройствах
2. Войти в одну группу
3. На первом устройстве использовать вето
4. **Ожидаемый результат**: На втором устройстве автоматически обновится:
   - Статус сессии
   - Количество токенов вето
   - История вето

## Известные ограничения

1. **Нет отображения истории вето** - пользователи не видят, кто и когда использовал вето (низкий приоритет)
2. **Нет восстановления токенов** - токены вето не восстанавливаются (по дизайну)
3. **Нет ограничения по времени** - вето можно использовать в любое время после решения (по дизайну)
