## Известные проблемы

### 1. Нельзя добавить больше одного варианта выбора
**Симптомы**: После добавления первого варианта, второй не добавляется

**Возможные причины**:
- Проблема с Realtime подпиской
- Проблема с обновлением состояния провайдера
- Ошибка в базе данных (проверить логи)

**Для отладки**:
1. Запустить приложение
2. Создать группу
3. Добавить первый вариант
4. Попробовать добавить второй вариант
5. Посмотреть логи в консоли (ищи строки с `DEBUG:`)

### 2. Нельзя удалить добавленный вариант
**Симптомы**: При нажатии на кнопку удаления ничего не происходит

**Возможные причины**:
- Проблема с Realtime подпиской
- Ошибка в базе данных
- Проблема с правами доступа

**Для отладки**:
1. Добавить вариант
2. Нажать на кнопку удаления (крестик)
3. Посмотреть логи в консоли

## Следующие шаги для отладки

### Шаг 1: Проверить логи
После добавления/удаления варианта в консоли должны появиться строки:
```
DEBUG: Начинаем добавление варианта: [название]
DEBUG: Текущая сессия: [id]
DEBUG: Добавляем вариант в существующую сессию
DEBUG: Вариант добавлен
DEBUG: Поле очищено
```

Или для удаления:
```
DEBUG: Удаляем вариант: [id] из сессии: [id]
DEBUG: Вариант удален успешно
```

### Шаг 2: Проверить Realtime в Supabase
1. Зайти в Supabase Dashboard
2. Перейти в Database → Replication
3. Убедиться, что Realtime включен для таблиц `sessions` и `options`

### Шаг 3: Проверить данные в базе
Выполнить SQL запрос:
```sql
-- Проверить сессии
SELECT * FROM sessions ORDER BY created_at DESC LIMIT 5;

-- Проверить варианты
SELECT * FROM options ORDER BY created_at DESC LIMIT 10;
```

### Шаг 4: Проверить Realtime подписку
Возможно, проблема в том, что Realtime не работает. Попробуй:
1. Перезапустить приложение
2. Создать новую группу
3. Добавить варианты
