# ТЕСТИРОВАНИЕ СИНХРОНИЗАЦИИ - ИНСТРУКЦИЯ

## Что проверить:

### 1. Дублирование вариантов
**Проблема:** После добавления варианта он дублируется

**Проверка:**
- Откройте приложение на двух устройствах
- Войдите в одну группу
- Добавьте вариант с первого устройства
- Проверьте, сколько раз он появился на обоих устройствах

**Ожидаемое поведение:** Вариант должен появиться ОДИН раз на обоих устройствах

### 2. Синхронизация участников
**Проблема:** Участники не синхронизируются или показываются без имен

**Проверка:**
- Второе устройство присоединяется к группе
- Проверьте, виден ли новый участник на первом устройстве
- Проверьте, отображается ли имя пользователя

**Ожидаемое поведение:** Новый участник должен появиться с именем на всех устройствах

### 3. Удаление вариантов
**Проверка:**
- Удалите вариант с одного устройства
- Проверьте, исчез ли он на втором устройстве

**Ожидаемое поведение:** Вариант должен исчезнуть на обоих устройствах

## Debug логи

Проверьте логи в консоли:
```
DEBUG: Начинаем добавление варианта: [название]
DEBUG: Вариант добавлен
DEBUG LOBBY: Session state changed
```

## Если есть проблемы:

1. **Дублирование** - проблема в primaryKey Stream или множественные подписки
2. **Нет синхронизации** - проблема в фильтрации `.eq('session_id', ...)` или `.eq('group_id', ...)`
3. **Нет имен участников** - Stream не загружает данные из таблицы users

Опишите подробно что происходит и я исправлю.
